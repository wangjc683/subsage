Calendar heatmap, cross-page drill-down, and a comprehensive UI/UX design audit with 11 fixes.

## ✨ New Features
- **Calendar Heatmap** — Days with subscriptions now show intensity-mapped green backgrounds (linear alpha 0.04–0.45), replacing the previous flat gray. Higher spending days are visually darker
- **Category Drill-Down** — Clicking a category row in the Overview "分类支出" section navigates directly to the Subscriptions page filtered by that category

## 📱 UX / UI
- **Hover-Reveal Actions** — Edit/delete buttons on subscription cards are now hidden by default and revealed on hover, reducing visual clutter on wide screens (P1)
- **Upcoming Renewal Connection** — Hover on renewal items adds a green left border accent for better visual grouping (P2)
- **Filter Separator** — Added a divider line between category and status filter rows (P3)
- **Agent Copy Button** — Downgraded from primary CTA to outlined secondary style (P4)
- **Trend Chart Line** — Average line increased from 2px to 2.5px with bolder dash pattern (P5)
- **Batch Button Hover** — Enhanced with primary color tint on hover (P6)
- **Card Shadows** — Increased `--shadow-sm` opacity for better card-background separation (P7)
- **Activity List Alternating** — Agent activity rows now have alternating background tints (P8)
- **Design Token Migration** — Replaced all hardcoded `border-radius` values across 8 files with CSS custom properties (P9)
- **Unified Pill Styles** — Status pills (Active/Paused/Cancelled) now match category pill styling with consistent borders and active states (P10)
- **Rate Chip Spacing** — Improved exchange rate chip gaps in Settings (P11)

## 🐛 Bug Fixes
- **Calendar Heatmap Reactivity** — Fixed Svelte reactivity chain so `maxDayAmount` correctly recalculates when subscription data changes
